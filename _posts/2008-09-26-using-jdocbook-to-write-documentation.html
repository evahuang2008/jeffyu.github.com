I used to use MS word to write articles, with a set of pre defined formats, such as Heading and so on, so that can produce a TOC (table of content). well, I still don't like to use Word to write articles. And then I find the JBoss Docbook is very good and easy to use. Use the JBoss Docbook is quite easy, now it is mavenized. you just simply define dependency and then add the task in the build process, that's all. BTW, if you are using Linux box, you can open the doc by "Open with Help" without running mvn install to build it. So personally, I use Eclipse, with Xmlbuddy plugin to write the document, and it works quite well.
